## 검토 배경

현재 `yahoo_finance_collector.py`는 **한국 테마 ETF 5종 거래량 급증**만 수집한다.
yfinance 라이브러리가 제공하는 다른 데이터들이 `raw_economic_data` (돈의 흐름)
도메인에 추가 가치를 주는지, 그리고 **거래량 시계열 자체의 수집 범위**를
넓혀야 하는지 적극 검토한다.

검토 방법:
- yfinance 1회성 프로브 (`scripts/_probe_yahoo_finance_coverage.py`) 로 실제 가용성 측정
- 각 데이터를 (도메인 적합성 / 선행성 / 구현 비용) 3축으로 평가
- Phase 1 즉시 반영 가능 항목 vs Phase 2 이후 이관 항목 분류

### 🟢 카테고리 B: 가격 변동 급증 (Price Surge) — 신규 알고리즘 / 신규 Collector

환율 / 국채 금리 / 원자재처럼 **거래량 개념이 없거나 무의미한 자산**을 위한
별도 알고리즘이 필요하다.

**알고리즘 제안** (Z-score 기반):
```
일간 수익률 r_t = (close_t / close_{t-1}) - 1
20일 일간 수익률 표준편차 σ_20 = std(r over last 20 trading days)
Z-score = |r_t| / σ_20
임계값: Z >= 2.0 (정규분포 가정 시 상위 ~5%)
```

| 그룹 | 티커 후보 | source_type | 알고리즘 | 추가 가치 |
|------|---------|-------------|---------|----------|
| **환율 (자본 국경 이동)** | `USDKRW=X` 원/달러 | `YAHOO_FX_USDKRW` | Z>=2.0 | **외국인 자본 유출입 핵심 신호** |
| | `EURKRW=X` 원/유로 | `YAHOO_FX_EURKRW` | Z>=2.0 | 유럽 자본 |
| | `JPYKRW=X` 원/엔 | `YAHOO_FX_JPYKRW` | Z>=2.0 | 일본 자본 |
| **금리 (자본 비용)** | `^TNX` 미 10년물 | `YAHOO_RATE_US10Y` | Z>=2.0 | **글로벌 무위험 수익률** |
| | `^IRX` 미 단기물 | `YAHOO_RATE_US3M` | Z>=2.0 | 단기 유동성 |
| **원자재 / 안전자산** | `GC=F` 금 | `YAHOO_COMMODITY_GOLD` | Z>=2.0 | 위험회피 자산 흐름 |
| | `CL=F` WTI 원유 | `YAHOO_COMMODITY_OIL` | Z>=2.0 | 산업 비용/인플레 |
| | `BTC-USD` 비트코인 | `YAHOO_COMMODITY_BTC` | Z>=2.5 (변동성 큼) | 위험자산 극단 |

**구현 비용**: 중. 새 알고리즘 함수 1개 + 새 Collector 클래스 1개.
**스키마 매핑**:
- `investment_amount`: 가격 자체로는 "유입액" 정의가 모호 → **`None`** 권장
- `raw_metadata`: 종가, 일간 수익률, 20일 σ, Z-score 모두 기록
- `raw_title`: "원/달러(USDKRW) 1.32% 급변 (Z=2.3, 2026-05-11)"

### 🟡 카테고리 C: 시총·PER 등 스냅샷 (Ticker.info) — Phase 2 이관

이건 "이벤트"가 아니라 **매일 변하는 상태값** 이라 `raw_economic_data`

## 6. 도메인 정합성 정리

**핵심 질문**: 이게 정말 `raw_economic_data` (돈의 흐름)에 맞는가?

| 데이터 | 도메인 정당화 |
|--------|--------------|
| 한국 대형주 거래량 급증 | 개별 종목으로 들어가는 직접적 자본 유입 신호 ✅ |
| 글로벌 ETF 거래량 급증 | 한국 시장의 **시간차 선행 지표** (미국 → 아시아 6~12시간) ✅ |
| 환율 (USDKRW 등) | **외국인 자본의 한국 시장 유출입**을 가장 직접 측정 ✅ |
| 미 국채 금리 변동 | **자본 비용(할인율) 변화**, 모든 자산 가격에 선행 ✅ |
| 금/원유/비트코인 변동 | 안전자산/위험자산 선호 변화 → 자본 이동 방향 ✅ |
| 옵션 체인 | 파생 포지션 = 미래 자본 흐름 베팅 (선행성 최고, 구현은 어려움) ✅ |
| 시총/PER (스냅샷) | "흐름"이라기보다 "상태값" — 트리거 이벤트로 변환 필요 ⚠️ |
| 재무제표 | 후행 지표 + DART 중복 ❌ |
| 배당/분할 | 후행 + 일회성 이벤트 ❌ |
| ETF Holdings | 한국 ETF 미지원 ❌ |
| 뉴스 | `raw_discourse_data` 영역 ❌ |

**결론**:
> **확장은 정당화된다.** 단, 무차별 확장이 아니라
> (1) **거래량 시계열 확장** (한국 대형주 + 글로벌 ETF) 과
> (2) **가격 변동 신규 Collector** (환율 / 금리 / 원자재) 두 축이 핵심이다.
> 재무제표·배당·옵션·뉴스 등은 도메인이 다르거나 가치가 낮다.

## 7. 다음 단계 옵션

### Option A — 카테고리 A만 즉시 적용 (보수적 빠른 적용, 추천)

> "현재 알고리즘 그대로, 티커만 11~16개로 늘린다"

작업:
1. `ETF_TARGETS` 에 한국 대형주 5종 + 글로벌 ETF 6종 추가
2. NaN 행 스킵 로직 추가
3. 티커 간 `asyncio.sleep(0.5)` 추가 (IP 차단 방어)
4. 통합 테스트 재실행

소요: 1~2시간. 즉시 데이터 커버리지 3배 확장.

### Option B — 카테고리 A + B 동시 적용 (포괄적)

작업: Option A + 신규 `YahooMacroCollector` (가격 변동 Z-score) 구현
- 신규 파일: `yahoo_macro_collector.py`
- 신규 서비스 메서드: `ingest_yahoo_macro()`
- 신규 라우터: `POST /api/master/bronze/economic/yahoo-macro`

소요: 반나절. **자본 흐름 선행 지표가 한 단계 진화** (환율/금리 포함).

### Option C — 검토 결과만 반영, 구현은 보류

검토 문서만 남기고, 다음 출처(The VC 등)부터 진행.
