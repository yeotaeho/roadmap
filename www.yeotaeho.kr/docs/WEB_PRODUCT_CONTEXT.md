# 웹 우선 개발 컨텍스트 (4탭 MVP)

## 문서 목적
- 이 문서는 새 채팅에서도 동일한 제품 컨텍스트를 빠르게 이어가기 위한 기준 문서입니다.
- 현재 단계에서의 우선순위, 범위, 구현 원칙을 명확히 고정합니다.

## 프로젝트 방향
- 본 서비스의 핵심은 **세상의 흐름(Macro)** 과 **개인의 성장(Micro)** 을 연결하는 것입니다.
- 구현 순서는 **웹 화면 완성 우선**이며, 앱은 웹이 충분히 안정화된 이후 별도로 진행합니다.

## 현재 합의된 개발 전략
- 1단계: 웹(`www.yeotaeho.kr`)에서 4개 탭 구조를 먼저 구현
- 2단계: 웹 기능/데이터 흐름/UX 검증 완료
- 3단계: 앱은 Flutter 등으로 신규 구현 (웹 로직 재사용 가능한 API 중심)

## 웹 MVP 정보구조 (4탭)
1. **인사이트 대시보드 (Global Pulse)**
   - 트렌드 속도계
   - 결핍 지도(Opportunity Map)
   - 3줄 경제 브리핑
   - 사용자 가치: 세상이 어디로 가는지에 대한 정보력

2. **AI 상담실 / 나의 발견 (Deep Discovery)**
   - 가치관 다이얼로그
   - 역량 레벨 테스트(AI 인터뷰)
   - 잠재력 매칭
   - 사용자 가치: 나의 성향/수준에 대한 객관성

3. **전략 로드맵 (Strategic Roadmap)**
   - 스킬 트라이앵글 로드맵(핵심 3스킬)
   - 포트폴리오 빌더 제안
   - 직무 키워드 브릿지
   - 사용자 가치: 당장 실행 가능한 방향성

4. **AI 코치 (Daily Mentor)**
   - 실시간 Q&A (트렌드/로드맵 연계)
   - 성장 로그(Insight Wallet)
   - 사용자 가치: 지속 실행력과 학습 루프

## 대시보드 세부 탭 전략 (L2: 4개 고정)
- 대시보드 내부는 아래 4개 세부 탭으로 고정하며, 이는 핵심 차별화 포인트입니다.
- 흐름 설계 원칙: **현황 파악 -> 기회 포착 -> 적합성 판단 -> 실제 행동**

1. **실시간 펄스 (Pulse)**
   - 구성(최신):
     - 글로벌 펄스 헤더(현재 속도, 전일 대비, 주간 지수, 한 줄 인사이트)
     - Top 섹터 히트맵/매트릭스(분야 x 상태, 색 강도 + 시그널 아이콘)
     - 인과관계 체인(거시 이벤트 -> 산업 영향 -> 청년 기회)
     - 세대교체/크로스오버 라인 차트(기존 수요 하락 vs 신흥 수요 상승)
     - 3줄 경제 브리핑 + 실시간 키워드 티커
   - 목표: 사용자가 5초 안에 "지금 분위기"를 파악

### 실시간 펄스 시각화 원칙 (운영 규칙)
- 펄스 탭은 **필터 중심 탐색 UI**가 아니라 **상황판(Big Picture) UI**로 설계합니다.
- 정보 계층은 다음 순서를 유지합니다:
  1) 상단 요약(헤더) -> 2) 관계/전환 신호(매트릭스/체인/크로스오버) -> 3) 실시간 피드(브리핑/티커)
- 카드별로 "한 줄 결론"을 명확히 노출해 해석 비용을 낮춥니다.
- 초기 릴리즈는 Mock 데이터를 허용하되, API 연동 시 동일한 데이터 형태(shape)를 유지합니다.

2. **블루오션 (The Gap)**
   - 구성: 세상의 결핍(구체적 문제 정의) + 기대 가치/보상
   - 목표: 막연한 불안을 "기회 인식"으로 전환
   - 메모: 킬러 콘텐츠로 우선 품질 확보

3. **싱크로율 (Sync)**
   - 구성: 매칭 시그널(역량-트렌드 일치도) + 관심사 변화 추이
   - 목표: "세상과 나의 연결"을 데이터로 증명

4. **다이렉트 찬스 (Chance)**
   - 구성: 맞춤 공고/공모전/지원사업 + 데드라인 알림
   - 목표: 분석 결과를 즉시 행동으로 연결
   - 메모: 킬러 탭, 실행 전환율 핵심

## 웹/앱 UX 통일 원칙 (강제)
- 웹과 앱의 정보구조(L1/L2)는 동일하게 유지합니다.
- 플랫폼별로 달라지는 것은 **레이아웃과 인터랙션 방식만** 허용합니다.
- 이유:
  - 사용자 학습 비용 최소화
  - 운영/유지보수 복잡도 감소
  - 웹 기반 앱 패키징(예: Capacitor) 대응 용이

## IA 계층 고정안 (웹/앱 공통)
- **L1 (메인):** 대시보드 / 코치 / 로드맵 / 지갑
- **L2 (대시보드 세부):** 펄스 / 블루오션 / 싱크 / 찬스

## 반응형 레이아웃 가이드
- **웹(데스크탑):**
  - L1은 사이드바 중심
  - L2는 상단 칩(Chip) 또는 아이콘 탭
  - 대시보드 핵심 지표는 위젯 카드로 요약 노출 후 상세 진입
- **앱(모바일):**
  - L1은 하단 탭바
  - L2는 상단 세그먼트 또는 가로 스크롤 탭
  - 엄지 동선 기준으로 1차 액션 버튼(저장/질문/지원)을 하단 고정

## UX 연결 규칙 (데이터 선순환)
- 블루오션에서 이슈를 찜하면:
  - 싱크로율 탭의 매칭 점수에 즉시 반영
  - 다이렉트 찬스 탭에 관련 기회 우선 노출
- 각 세부 탭 하단에 공통 CTA를 둡니다:
  - "AI 코치에게 이 내용 물어보기" -> 코치 탭 이동

## 구현 시 주의사항
- 탭 개수를 줄여 임시 대응하지 않습니다. (웹/앱 불일치 금지)
- 복잡해 보이는 문제는 탭 축소가 아니라 시각 계층/배치로 해결합니다.
- 초기 릴리즈는 탭별 대표 카드 1~2개부터 시작하고 점진 확장합니다.

## 우선 구현 원칙 (중요)
- 한 번에 모든 기능을 완성하지 않고, **탭별 1개 핵심 기능부터** 완성합니다.
- 탭 간 연결은 CTA로 강하게 설계합니다.
  - 예: 인사이트 저장 -> 프로필 반영 -> 로드맵 생성 -> 오늘의 액션 실행
- 디자인보다 **데이터 흐름의 일관성**을 먼저 확보합니다.

## 권장 1차 구현 순서
1. 인사이트 대시보드 실데이터 연결 (기존 뉴스/트렌드 데이터 활용)
2. 전략 로드맵 MVP (현재 역량 + 선택 트렌드 -> 3개 액션 제안)
3. AI 코치(로드맵 컨텍스트 기반 Q&A)
4. 딥 디스커버리 고도화(프로파일링 정밀화)

## 웹/앱 분리 원칙
- 앱은 웹의 UI를 단순 이식하지 않고, 모바일 사용성 기준으로 재설계합니다.
- 앱 개발 시작 조건:
  - 웹 4탭 기본 플로우 완성
  - 핵심 API 스펙 안정화
  - 사용자 테스트로 핵심 가치 검증 완료

## 기술 방향 메모
- 프론트엔드: Next.js 기반 4탭 UX 고도화
- 백엔드: FastAPI 중심 API 제공
- 에이전트/오케스트레이션: LangGraph 기반 상태 전이
  - 상담(State: User Profile) -> 분석(State: Trend Data) -> 계획(State: Roadmap)
- 배치/수집: 스케줄러 기반 데이터 업데이트 파이프라인
- 펄스 실데이터 API 우선순위(권장):
  - `/pulse/summary` (속도, 전일 대비, 주간 지수, 인사이트)
  - `/pulse/matrix` (분야 x 상태 셀 점수/시그널)
  - `/pulse/causal-chain` (거시 -> 산업 -> 기회)
  - `/pulse/crossover` (기존/신흥 수요 시계열)
  - `/pulse/ticker` (실시간 키워드 스트림)
