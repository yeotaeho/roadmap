# LLM 컨텍스트 인계 문서 (MVP 진행 기준)

## 1) 문서 목적
- 이 문서는 새 LLM 세션이 프로젝트 맥락을 빠르게 이해하고, 바로 구현/개선 작업을 이어가기 위한 인계 문서다.
- 제품 철학, IA, 현재 구현 상태, 남은 작업 우선순위를 한 번에 제공한다.

---

## 2) 프로젝트 한 줄 정의
- **AI 기반 청년 글로벌 통찰력 서비스(MVP)**: 청년의 막연한 진로 불안을 줄이기 위해, **세상 흐름(Macro)** 과 **개인 성장(Micro)** 을 연결하는 데이터 기반 내비게이션을 제공한다.

## 3) 핵심 목표와 가치
- 목표: 청년이 "지금 무엇을 준비해야 하는지"를 객관적 지표로 이해하고 실행하게 만든다.
- 핵심 가치:
  - 단순 직업 추천이 아닌 **나의 잠재력 x 세상의 요구** 연결
  - 트렌드 해석 -> 개인 적합도 -> 실행 액션으로 이어지는 루프

## 4) 대상 사용자
- 10대 후반 ~ 30대 초반
- 진로 방향이 불명확하거나, 시장 변화에 대한 구조적 이해가 필요한 청년/예비 사회인

---

## 5) 정보구조(IA) 고정안

### L1 (메인 4탭)
1. 인사이트 대시보드
2. AI 상담실
3. 전략 로드맵
4. AI 코치

### L2 (대시보드 세부 4탭)
1. 실시간 펄스 (Pulse)
2. 블루오션 (The Gap)
3. 싱크로율 (Sync)
4. 다이렉트 찬스 (Chance)

### 설계 원칙
- 흐름: **현황 파악 -> 기회 포착 -> 적합성 판단 -> 실제 행동**
- 웹/앱은 IA 동일, 차이는 레이아웃/인터랙션만 허용

---

## 6) 현재 코드 구현 상태 (중요)

## 6.1 라우팅/레이아웃
- 메인 엔트리: `/`에서 대시보드 렌더
- 메인 탭 구조 존재(인사이트/상담/로드맵/코치)
- 파일:
  - `src/app/(main)/page.tsx`
  - `src/components/layout/MainTabBar.tsx`
  - `src/components/layout/Header.tsx`

## 6.2 대시보드 L2 4탭
- 탭 전환 UI 및 4개 패널 골격 구현 완료
- 파일:
  - `src/components/features/dashboard/DashboardView.tsx`

## 6.3 실시간 펄스(Pulse) 탭
- 사용자 제공 JSX + `gemini-code-1777040200243.md` 명세 반영 완료
- 최신 반영 요소:
  - 글로벌 펄스 헤더(현재 속도, 전일 대비, 주간 지수, 한 줄 인사이트)
  - Top 섹터 히트맵/매트릭스(분야 x 상태, 색 강도 + 기회/리스크 아이콘)
  - 인과관계 체인(거시 이벤트 -> 산업 영향 -> 청년 기회)
  - 세대교체/크로스오버 라인 차트(기존 수요 하락 vs 신흥 수요 상승)
  - 3줄 경제 브리핑 + 실시간 키워드 티커
  - Top 6 섹터 속도 카드, 연간 모멘텀, 점유율, 키워드 클라우드
- 파일:
  - `src/components/features/dashboard/PulseTab.tsx`
  - 명세 문서: `docs/gemini-code-1777040200243.md`

## 6.4 나머지 탭 상태
- 블루오션/싱크/찬스: UI 컨셉은 구현, 데이터는 Mock 중심
- AI 상담실/전략 로드맵: 소개/placeholder 성격 강함
- AI 코치: 채팅 UI 있음, 응답은 Mock 로직

---

## 7) 데이터/엔진 관점 현재 상태
- 프론트 중심 MVP로 진행 중이며, 다수 지표는 현재 Mock 데이터
- 향후 방향:
  - FastAPI 기반 API 연동
  - 룰 기반 점수화 엔진(검색량/언급량/투자/공고 등 가중치)
  - LangGraph 오케스트레이션(상담 -> 분석 -> 계획)

---

## 8) 즉시 우선순위 (실행 순서)
1. 대시보드 실데이터 연결 (Pulse 5요소 API부터 우선)
2. 개인화 매칭(사용자 성향/관심 -> Top 3 트렌드 하이라이트)
3. 로드맵 생성 API(트렌드 선택 -> 핵심 역량 3개 + 실행 액션)
4. AI 코치에 컨텍스트 주입(RAG/요약 기반 답변 품질 고도화)

### Pulse API 권장 분리
- `/pulse/summary`
- `/pulse/matrix`
- `/pulse/causal-chain`
- `/pulse/crossover`
- `/pulse/ticker`

## 9) LLM 작업 규칙 (이 문서를 읽는 모델용)
- 본 프로젝트는 **웹 우선 MVP**다. 앱 전용 구현 제안은 보류한다.
- 탭 수/IA를 임의 축소하지 않는다.
- UI 확장보다 **데이터 흐름 일관성**을 우선한다.
- 변경 제안 시 아래 순서를 지킨다:
  1) 현재 상태 진단
  2) 영향 파일 명시
  3) 최소 변경 구현
  4) 빌드/린트 확인
