# 화면장수·정의 및 설계서 (인사이트 대시보드 + AI 상담실 + 전략 로드맵 + AI 코치)

## 문서 목적

- 대시보드 L2 4탭(실시간 펄스, 블루오션, 싱크로율, 다이렉트 찬스) 기준으로 **화면을 몇 장으로 쪼갤지**를 고정합니다.
- AI 상담실(`/consult`)의 최신 UI 구조(실시간 채팅 캔버스 + 라이브 결과 패널)를 문서화하여 구현/QA 기준을 통일합니다.
- 전략 로드맵(`/roadmap`)의 **L1 단일 라우트 + L2 서브탭(여정 개요 / 성장 아카이브)** 구조, 화면장수 규칙, 데이터·컴포넌트 매핑을 문서화합니다.
- AI 코치(`/coach`)의 2열 레이아웃(대화 캔버스 + 인사이트 월렛)·모바일 UX를 문서화합니다.
- 전역 셸(헤더·메인 탭바·푸터·최대 폭·다크 모드·프라이머리 컬러) 기준을 한곳에 묶어 디자인·QA 기준을 맞춥니다.
- 각 화면의 **목적, 핵심 UI, 데이터, 다음 작업(구현 순서)**을 명확히 해서 프론트/백엔드/기획이 같은 기준으로 움직이게 합니다.
- 현재 코드 기준(Next.js App Router)에서 바로 이어갈 수 있도록 **권장 라우트**까지 제시합니다.

### 화면장수 정리(역할별)

| 범위 | 장수 규칙 | 숫자 |
| :--- | :--- | :---: |
| 대시보드 L2 탭 자체 | URL 1개(`/`) 안의 4개 뷰 | **4「탭」** (URL 가산 없음) |
| 대시보드 확장 상세 | 독립 라우트 | **8장** |
| **소계(기존 합의「12장」)** | L2 4 + 상세 8 | **12** |
| L1 추가(상담·로드맵·코치) | 각각 독립 URL | **+3장** (`/`와 합치면 L1은 4URL) |
| 인증·프로필 등 | 제품 코어 IA 밖 | 문서 별도 표기 |

> **참고**: 「12장」은 여전히 **대시보드 L2 범위만** 계산한 값입니다. 전체 서비스의 **라우트 수**는 로그인·상담실·로드맵·코치·OAuth 등을 포함하면 더 많습니다.

## 용어 정리

- **L1 탭**: 메인 4탭(인사이트 대시보드, AI 상담실, 전략 로드맵, AI 코치)
- **L2 탭**: 대시보드 내부 4탭 — 본 문서의 핵심 범위.
- **로드맵 서브탭(L2-로드맵)**: `/roadmap` **동일 URL** 안에서 전환되는 뷰 2종(여정 개요, 성장 아카이브). 대시보드의 “싱크 패널을 같은 탭 안에 넣기”와 동일하게 **총 URL 장수에는 가산하지 않음**(아래 전략 로드맵 절 참조).
- **화면 1장**: URL(라우트) 단위로 독립 진입 가능한 페이지, 또는 동일 URL에서 모달/드로어로 “독립 정보 구조”를 갖는 경우(권장은 URL 분리).

> 참고: 화면장수 “12장”은 **대시보드 L2 범위만** 계산합니다. AI 상담실·전략 로드맵은 **별도 범위**로 아래에 정의합니다.

## 화면장수(총 12장) — 확정안

### A) L2 탭 자체(4장)

1. 실시간 펄스 탭(목록/요약 화면)
2. 블루오션 탭(목록/요약 화면)
3. 싱크로율 탭(목록/요약 화면)
4. 다이렉트 찬스 탭(목록/요약 화면)

### B) 탭별 상세(8장)

5~10. 실시간 펄스: **도메인(섹터) 6개 상세** (각 1장)
11. 블루오션: **이슈 상세 1장** (카드의 “이 분야 파고들기” 진입)
12. 다이렉트 찬스: **기회 상세 1장** (카드의 “자세히 보기” 진입)

### C) 싱크로율 탭의 “추가 1장”에 대한 처리(장수 유지 원칙)

요청하신 구조는 “싱크로율 탭에서 성장 궤적 아래에 싱크 데이터 칸을 추가”입니다. 이는 **같은 탭 화면(3번) 내부의 섹션 확장**으로 처리하여 **별도 URL을 추가하지 않고** 총 장수를 **12장**으로 유지합니다.

> 장수를 늘리고 싶다면(13장 이상): `/sync/trends` 같은 “싱크 상세”를 별도 라우트로 분리하면 됩니다. (현재 합의: 12장 고정)

## 정보구조(IA) — 대시보드 L2

### 실시간 펄스

- **탭 목적**: 5초 안에 “지금 세상의 속도/방향”을 조망.
- **핵심 데이터**: 도메인별 속도/상태, 모멘텀, 키워드, 인과 체인(대표 1개), 티커.

### 블루오션

- **탭 목적**: 막연한 불안을 “구체적 결핍 + 기회”로 전환.
- **핵심 데이터**: 문제 정의, 기회 서사, 근거 링크/지표(추후).

### 싱크로율

- **탭 목적**: “나 ↔ 트렌드”의 정합성을 수치/추세로 증명.
- **핵심 데이터**: 선택/저장한 관심 키워드, 트렌드 대비 싱크 점수, 변화 추이, 해석 문장.

### 다이렉트 찬스

- **탭 목적**: 분석을 즉시 행동(지원/공모/교육/채용)으로 연결.
- **핵심 데이터**: 기회 메타(타입/D-day), 매칭 점수, 상세 조건, 링크/저장.

## AI 상담실(별도 범위) 화면 정의

### 0) AI 상담실 메인 — 실시간 인터랙션 캔버스 (1장)

- **URL(권장)**: `/consult`
- **화면 목적**: 유저 응답에 따라 다음 질문이 실시간으로 바뀌는 동적 인터뷰를 수행하고, 우측 패널에서 분석 결과를 즉시 피드백.
- **현재 구현 컴포넌트**: `src/app/(main)/consult/page.tsx`

#### 레이아웃

- **2-Column Grid**
  - 좌측 65%: Interactive Zone (실시간 메시지 피드 + 입력창)
  - 우측 35%: Live Result Panel (진행률 + 레이더 + 키워드 + CTA)
- 톤앤매너: 인디고(`#4F46E5`) + 슬레이트 계열 배경·텍스트; **다크 모드** 시 `dark:` 변형으로 AI 코치 탭과 톤 통일

#### 좌측(Interactive Zone) 규격

- **메시지 피드(`overflow-y-auto`)**
  - AI 메시지: 좌측 정렬, `bg-slate-100`
  - User 메시지: 우측 정렬, `bg-indigo-600 text-white`
  - System 메시지: 중앙 정렬 pill(단계 전환 안내)
  - 역량 질문은 텍스트 버블 외에 카드형(`rich=card`) 표현 가능
- **입력 영역(하단 고정)**
  - 텍스트 입력 + 엔터 전송 + 전송 버튼
  - Quick Reply 칩(객관식) 지원
  - 하이브리드 입력 정책:
    - 가치관 단계: Quick Reply 우선
    - 역량 단계: 서술형 입력 우선

#### 우측(Live Result Panel) 규격

- **상단**: 전체 진행률 바(`phase` 기반)
- **중단**: `recharts` 레이더 차트(턴 진행 시 애니메이션 갱신)
- **하단**: `extracted_keywords` 라이브 태그 클라우드
- **요약 카드**: 성향 한 줄 + 액션 CTA(로드맵/Chance/AI 코치)

#### 상태 전이(목업 기준)

- `phase 1` 가치관 대화
- `phase 2` 역량 시나리오
- `phase 3` 리포트 요약
- 단계 전환 시 시스템 메시지 삽입 및 우측 패널 수치 동기화

#### 데이터 연계(백엔드 ERD 매핑)

- 세션 시작: `consultation_sessions`
- 턴 저장: `consultation_turns` (`psych_analysis`, `tech_analysis`, `routing_decision`)
- 누적 성향: `user_personas`
- 누적 역량: `user_competencies`

## AI 코치(별도 범위) 화면 정의

### 0) AI 코치 메인 — 대화 캔버스 + 인사이트 월렛 (1장)

- **URL**: `/coach`
- **화면 목적**: 로드맵·찬스 등 **맥락을 붙인 코칭**과, 코칭 중 나온 스니펫·요약을 **Insight Wallet**에 쌓아 두는 실행형 멘토링 UI.
- **루트 컴포넌트**: `src/app/(main)/coach/page.tsx` → `CoachView`
- **우측 패널 컴포넌트**: `src/components/features/coach/InsightWalletPanel.tsx`

#### 레이아웃

- **데스크톱**: 좌측 메시지 영역 + 우측 **Active Context**(현재 로드맵 포커스 등) 및 **Insight Wallet**(저장 항목).
- **모바일**: 우측 패널은 **FAB / 바텀시트**로 동일 정보 접근(코치 화면 일관성 유지).
- **폭**: `CoachView` 루트에 별도 `max-w-*` 제한 없음 → `MainLayout`의 **`max-w-[1480px]`** 과 다른 L1 탭과 동일.

#### 좌측(대화 영역)

- 사용자·어시스턴트 메시지 버블, 필요 시 **코드 블록**(예: 룰 기반 스코어링 예시).
- **프로액티브 첫 인사**: 로드맵 연계 질문 형태의 데모 메시지(`buildProactiveGreeting`).
- 입력: 전송 버튼·키보드 전송; 목업 응답은 키워드·첨부 컨텍스트에 따라 분기(`mockReply`).

#### 우측(Insight Wallet 패널)

- **활성 컨텍스트**: 데모 데이터 `COACH_ACTIVE_FOCUS`, `DEMO_ATTACHED_CONTEXTS`.
- **월렛**: 저장된 인사이트 목록(데모); 「월렛에 저장」류 액션과 연계 가능한 구조.

#### 데이터 소스(프론트 데모)

- **파일**: `src/data/coachContext.ts`
- **백엔드 연계 시**: 세션/턴·월렛 아이템 영속화, 로드맵·찬스와의 FK 또는 이벤트 동기화가 필요.

## 전략 로드맵(별도 범위) — 화면장수·IA·구현 매핑

### 화면장수(로드맵) — 확정 규칙

| 구분 | 장수( URL 기준 ) | 설명 |
| :--- | :---: | :--- |

- **하단 CTA**: AI 상담실, AI 코치, 인사이트 대시보드 링크

### [탭 1] 여정 개요 (Journey Map)

- **컴포넌트**: `src/components/features/roadmap/JourneyMapTab.tsx`
- **섹션 구성**
  1. **스킬 트라이앵글**: 핵심 3축(`SKILL_TRIANGLE`) + 중앙 `YOU` — 역량 **방향** 고정(마감·진행률 바 강제 없음)
  2. **직무 키워드 브릿지**: `BRIDGE_KEYWORDS` 칩 — 대시보드·상담실과의 **시맨틱 연결 앵커**
  3. **퀘스트 트리 (Quest Tree)**: `QUEST_TREE` 재귀 렌더 — `QuestTreeNode`의 `children`으로 분기, 왼쪽 세로 맥락선 + 카드
- **퀘스트 카드에 넣은 메타(일정 압박 최소화)**
  - `title`, `purpose`, `difficulty`(입문/중급/심화), `keywords[]`, `state`(start | available | active | done | locked)
- **의도적으로 제외한 것**: 칸반(To-do / In Progress / Done), 대형 Gantt형 일정 바, 스프린트 진행률 위젯(이전 구현에서 제거됨)

### [탭 2] 성장 아카이브 (Growth Calendar)

- **컴포넌트**: `src/components/features/roadmap/GrowthArchiveTab.tsx`
- **좌측 — 달력**
  - 월 이동(이전/다음 달)
  - 해당 월 그리드(일~토 헤더)
  - `logs`에 기록 또는 달성 퀘스트가 있으면 **점(민트/인디고)** 표시(잔디/캘린더 하이라이트 느낌)
  - 오늘 날짜·선택 날짜 시각 구분
- **우측 — Daily Log 패널**
  - 선택일 표기(로케일 `ko-KR`)
  - **이날 달성한 퀘스트**: `QUEST_TREE`를 `flattenQuestTitles`로 펼친 목록 기준 체크박스( `completedQuestIds` )
  - **자유 기록**: `textarea`(마크다운 스타일 자유 입력 가정; 렌더는 현재 평문)
  - **저장**: 현재는 **클라이언트 state만 갱신**(목업). 추후 `PUT` API로 치환.
- **시드 데이터**: `ARCHIVE_ACTIVITY_SEED` (`YYYY-MM-DD` → `{ completedQuestIds, note }`)

### 데이터 소스(프론트)

- **파일**: `src/data/roadmapQuestMap.ts`
- **타입 요약**
  - `SkillPillar`, `QuestTreeNode`, `QuestDifficulty`, `QuestTreeState`
  - `flattenQuestTitles(node)` — 아카이브 체크리스트용 플랫 리스트
- **백엔드 연계 시 권장 모델(문서 수준)**  
  - `roadmap_snapshots`(유저별 퀘스트 트리 JSON), `growth_day_logs`(date, user_id, note_md, completed_quest_ids[])

### 권장 API(초안)

- `GET /roadmap/journey` — 스킬 트라이앵글 + 브릿지 + `QUEST_TREE` 동등 JSON
- `GET /roadmap/logs?year=2026&month=4` — 월별 활동 요약(캘린더 점)
- `GET /roadmap/logs/{date}` — 일별 상세
- `PUT /roadmap/logs/{date}` — 노트 + 달성 퀘스트 id 목록 저장

## 보안/운영에서 반드시 챙길 부분

### 인증/인가

- 기회 상세/저장/알림은 로그인 사용자 기준으로 권한 분리
- 관리자 전용 콘텐츠(이슈 큐레이션)가 생기면 RBAC 필요

### 외부 링크/스크래핑

- 공고/뉴스는 가능한 한 **공식 URL로 outbound** (저작권/약관 리스크 최소화)
- 사용자 생성 텍스트(코멘트/메모) 도입 시 XSS 방지(렌더링 규칙)

### 개인정보/프로파일링

- 싱크로율은 민감할 수 있으니 “점수 설명/근거/삭제” UX를 초기부터 포함
- 로그에 토큰/PII가 남지 않게 마스킹

### 남용 방지

- 티커/키워드 API는 rate limit
- 크롤링 기반 데이터는 캐시/출처 표기 의무화
