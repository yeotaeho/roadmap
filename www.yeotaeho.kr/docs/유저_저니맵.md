# 유저 저니맵 (AI 상담실 · Deep Discovery)

## 문서 목적

- AI 상담실(`/consult`)에서 유저가 실제로 겪는 흐름을 **행동/감정/시스템 반응/데이터 축적** 관점으로 한 장에 정리한다.
- 화면 기획, API 설계, QA 체크를 같은 기준으로 맞춘다.

---

## 핵심 여정 (요약)

1. 진입: 메인 탭에서 `AI 상담실` 클릭
2. 시작: 세션 생성 및 안내 메시지 확인
3. 1단계 가치관 대화: Quick Reply 중심 인터랙션
4. 2단계 역량 인터뷰: 서술형 답변 중심 인터랙션
5. 3단계 결과 확인: 진행률/레이더/키워드/요약 리포트 확인
6. 전환 액션: 로드맵 생성, Chance 탐색, AI 코치 연계

---

## 단계별 상세 저니

### 0) 진입 전

- **유저 행동**
  - 대시보드/메인 화면에서 “AI 상담실” 탭 선택
- **유저 기대/감정**
  - “내가 어디에 맞는지 빠르게 알고 싶다”
- **시스템 반응**
  - 상담실 랜딩 렌더, 최근 상태가 있으면 이어하기 컨텍스트 준비
- **데이터**
  - (선택) 마지막 세션 조회

### 1) 세션 시작 (Onboarding Prompt)

- **유저 행동**
  - 첫 안내 메시지 확인
- **유저 기대/감정**
  - 부담 없이 시작 가능해야 함
- **시스템 반응**
  - `consultation_sessions` 생성 (`session_status=active`)
  - 좌측 채팅 캔버스 + 우측 라이브 패널 초기화
- **데이터**
  - `consultation_sessions.id`, `user_id`, `current_graph_node`, `started_at`

### 2) 가치관 탐색 (Phase 1)

- **유저 행동**
  - Quick Reply 칩(예: 안정적인 대기업/성장하는 스타트업) 선택
  - 필요 시 짧은 서술 입력
- **유저 기대/감정**
  - 빠르고 가벼운 응답 경험, 질문 의도 명확성
- **시스템 반응**
  - 질문-응답 턴 저장
  - 감정/가치 신호 추출 후 키워드 태그 업데이트
  - 우측 진행률/레이더 소폭 갱신
- **데이터**
  - `consultation_turns.psych_analysis`
  - `consultation_turns.routing_decision` (다음 질문 난이도/분기)

### 3) 역량 인터뷰 (Phase 2)

- **유저 행동**
  - 시나리오 질문(장애 대응, 우선순위 판단 등)에 서술형 답변
- **유저 기대/감정**
  - 실제 실무 맥락에서 평가받는 느낌
- **시스템 반응**
  - 카드형 AI 질문 출력
  - 응답 기반 기술 스택/깊이/도메인 신호 분석
  - 단계 전환 시스템 메시지 노출
- **데이터**
  - `consultation_turns.tech_analysis`
  - `consultation_turns.llm_reasoning` (디버깅/설명 근거)

### 4) 결과 요약 (Phase 3)

- **유저 행동**
  - 우측 라이브 리포트 확인(진행률/레이더/키워드/요약 문장)
- **유저 기대/감정**
  - “왜 이런 결과가 나왔는지” 이해 가능성
- **시스템 반응**
  - 누적 성향/역량 갱신
  - 다음 행동 버튼(로드맵/Chance/AI 코치) 노출
- **데이터**
  - `user_personas` upsert
  - `user_competencies` upsert + `evidence_log` 갱신

### 5) 전환 액션

- **유저 행동**
  - 로드맵 생성 또는 Chance 확인 또는 AI 코치로 이동
- **유저 기대/감정**
  - 분석이 실제 행동으로 이어지길 기대
- **시스템 반응**
  - 선택한 목적지로 라우팅 및 컨텍스트 전달
- **데이터**
  - (선택) 전환 이벤트 로깅, 세션 종료 처리(`completed`)

## 핵심 UX 원칙

- **하이브리드 입력**: 가치관은 Quick Reply, 역량은 서술형 중심
- **실시간 피드백**: 우측 패널이 턴마다 변해 “분석이 살아있다”는 감각 제공
- **설명 가능성**: 키워드/근거/단계 전환 메시지로 결과 납득성 확보
- **저마찰 전환**: 결과 확인 후 바로 행동(로드맵/기회/코치)으로 연결

---

## 실패 지점 및 대응

- 칩만 강제되어 답답함 -> “기타 직접 입력” 항상 허용
- 질문 길이 과다 -> 카드형 요약 + 핵심 질문 1문장 유지
- 결과 불신 -> 근거 턴(`evidence_log`) 기반 설명 문구 제공
- 이탈 증가 -> 단계별 소요 시간/남은 단계 명확히 표시

---

## QA 체크리스트 (빠른 점검)

- `/consult` 진입 시 세션이 1회 생성되는가
- 1단계에서 Quick Reply가 노출되고 동작하는가
- 2단계에서 서술형 입력 + 엔터 전송이 동작하는가
- 단계 전환 시스템 메시지가 보이는가
- 우측 진행률/레이더/키워드가 턴 진행과 함께 갱신되는가
- CTA 클릭 시 `/roadmap`, `/`, `/coach`로 정상 이동하는가
